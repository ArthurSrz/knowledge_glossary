Editors from Harvard Data Science Review wonder why using a data [[Design]] approach can be more powerful than using [[data visualization]] . My opinion is that data design is universal compared to vizualisations but I don't have examples in mind. Claude thinks vizualization operate at a [[Perception]] level while our data design approach operate at a data object structural level

Editors keep pointing into the direction that the paper submitted is hard to distinguish from [[data visualization]] techniques. But data vizualisation bakes in an incredible amount of choices by the author that cover much intelligence that would need to be known and reused. And it tells something about the data generation process ([[Synthetic data generation]]) that opens many opportunities for deeper statistical analysis as we one how the dataset was built (see [[Bayesian inference]], [[Bayesian network]] and [[causal inference]]). 

But the greatest challenge raised by the editors is : "why care so much now about these complexity issues now we have MCP and LLMs"...and that's a big one. Why should we still care about data complexity and mess when Claude seems to be able to make sense out of everything. I think the answer is once again the same :  Claude can make sense of it **for 1 person**, but this does not support the collective effort of having a standard way to deconstruct and reconstruct data objects, and making sense for the mess for everybody. I think it opens ways to create new taxonomies and new data structures in knowledge domain not yet having [[Metadata]] catalogs or this type of stuff.
