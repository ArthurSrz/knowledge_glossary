Comment faire un article didactique à partir d'une *release note* en utilisant simplement des inférences ? 

Tout d'abord, j'ajoute la *release note* dans cette note, et je laisse le [live anchoring](https://community.obsidian.md/plugins/live-anchoring) faire son travail. J'obtiens : 

> [!note] 
> - SME Homepage — a new Project Home [[dashboard]] with role-based views: interview queue, [[feedback]]/signal counts, pending invites, and an admin-only changelog widget
> - Work with Clara — a new fast, direct chat experience with Clara, with suggested quick-start actions, in-chat interview scheduling, and a [[knowledge]] changelog view
> - [[Design]] template artifacts — create, edit, version, and attach files to [[design]] [[templates]] directly in the project
> - New playground [[agent]] (v2) — a [[feature]]-flagged next-[[generation]] playground assistant with stronger [[context]] handling and citations, now the [[default]] for new projects (CLA-1453)
> - LiteLLM AI gateway support — LLM calls can now be routed per-[[model]] through a shared gateway, for both pipeline extraction and the [[backend]] playground
> - SharePoint: pick specific (non-[[default]]) document libraries to sync, admins can restrict imports to an allowlist of approved SharePoint sites, and downloads now run in parallel for faster imports
> - Staff-only Organizations admin in crfadmin — list, [[search]], and create organizations for support purposes
> - Sources page now shows each document's creation date and uploader, with sortable columns
