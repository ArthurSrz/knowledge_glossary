# Task Flow Diagram

## Definition

A task flow diagram is a graphical representation of the sequence of activities required to complete a process, originated by Frank and Lillian Gilbreth in 1921 as "process charts" to find "the one best way to do work."

## Historical Development

1. **Gilbreths' Process Charts (1921)**: First systematic method
2. **ASME Presentation**: "Process Charts: First Steps in Finding the One Best Way to do Work"
3. **Industrial Engineering (1930s)**: Widespread adoption
4. **Computer Science (1960s)**: Adapted for programming
5. **UX Design (1990s+)**: User experience workflows

## Gilbreths' Original Concept

According to Gilbreth & Gilbreth (1921):
- Systematic documentation of process flow
- Visual representation of work activities
- Tool for motion study and efficiency
- Method to identify waste and redundancy
- Foundation for process improvement

## Key Components

1. **Symbols**:
   - Operations (circle)
   - Inspections (square)
   - Transportation (arrow)
   - Delays (D-shape)
   - Storage (triangle)

2. **Flow Lines**:
   - Sequential connections
   - Decision points
   - Parallel processes
   - Feedback loops

3. **Annotations**:
   - Time durations
   - Resource requirements
   - Decision criteria
   - Performance metrics

## Types of Flow Diagrams

1. **Process Flow Chart**:
   - Linear sequence
   - Operation focus
   - Time analysis
   - Resource allocation

2. **Workflow Diagram**:
   - Multiple participants
   - Handoffs and transitions
   - Role responsibilities
   - Cross-functional processes

3. **Task Analysis Diagram**:
   - User-centered design
   - Cognitive tasks
   - Decision points
   - Error paths

4. **Swimlane Diagram**:
   - Departmental responsibilities
   - Cross-functional workflows
   - Process ownership
   - Interface points

## Design Principles

1. **Clarity**:
   - Simple symbols
   - Consistent notation
   - Logical flow
   - Minimal crossings

2. **Completeness**:
   - All steps included
   - Decision logic shown
   - Exception handling
   - Start/end points

3. **Accuracy**:
   - Verified processes
   - Correct sequence
   - Validated timing
   - Resource requirements
